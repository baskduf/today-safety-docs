# Today Safety Frontend Planning

## 목적

이 폴더는 Today Safety 프론트엔드 MVP를 설계하기 위한 문서를 모아두는 공간이다.

프론트엔드는 `AI 챗봇 화면`이 아니라, `현장관리자가 작업 전 3분 안에 브리핑 초안을 만드는 업무 도구`를 목표로 한다.

## 제품 관점 한 줄 정의

`오늘 작업 입력 -> 위험요약 생성 -> 체크리스트/브리핑 확인 -> 관리자 검토 -> 저장/PDF/공유`

## 핵심 UX 원칙

- 입력은 짧게 유지한다.
- 결과는 현장에서 바로 읽을 수 있어야 한다.
- 근거는 접어서 두고 필요할 때 펼쳐 확인하게 한다.
- 최종 판단은 관리자 검토와 승인 흐름에서 이뤄진다.
- 화면은 대시보드보다 `작업 전 브리핑 생성기`에 가깝게 설계한다.

## MVP 범위

- 메인 진입은 `/new` 중심으로 잡는다.
- 생성 결과는 `위험요약`, `사고유형 Top 3`, `체크리스트`, `브리핑 문안`, `근거 사례`, `검토 상태`로 고정한다.
- 초기 릴리스는 `새 브리핑`, `결과 상세`, `브리핑 기록`을 우선 구현한다.
- `PDF 저장`, `승인 상태`, `날씨 자동 조회`는 2순위 기능으로 둔다.

## 권장 페이지 구조

- `/`
  - 서비스 소개 또는 `/new`로 리다이렉트
- `/new`
  - 새 브리핑 생성
- `/result/:id`
  - 생성 결과 상세
- `/history`
  - 과거 브리핑 목록
- `/history/:id`
  - 과거 브리핑 상세
- `/settings`
  - 사업장 기본값, 기본 산업군, 위치 설정

## 구현 기준선

- 프레임워크: Next.js + TypeScript
- 렌더링 전략: App Router 기준 서버 렌더링 + 클라이언트 상호작용 혼합
- 데이터 연결: 우선 `POST /brief-draft-preview` 중심으로 시작
- 상태 관리: 브리핑 생성 폼과 결과 상태를 단순하게 유지하고 과도한 전역 상태는 피한다

## 비목표

- 챗봇형 자유대화 UI
- 운영 통계 대시보드 우선 개발
- 복잡한 멀티스텝 관리자 백오피스
- 정밀 사고확률 숫자 시각화 중심 화면
