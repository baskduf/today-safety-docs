# Taxonomy Review Guide

이 폴더는 사람이 `기타`로 남은 사고유형과 작업공정을 빠르게 검수할 수 있도록 만든 검수 전용 파일 모음입니다.

## 먼저 열 파일

- 검수용 엑셀:
  - `taxonomy_review_workbook.xlsx`
- 원본 CSV:
  - `accident_taxonomy_review.csv`
  - `process_taxonomy_review.csv`
- 빠른 우선순위 가이드:
  - `priority_review_guide.md`

가능하면 엑셀 파일부터 보는 것을 권장합니다. 시트가 나뉘어 있고, 헤더 고정과 필터가 쉬워서 훨씬 편합니다.

## 파일별 의미

- `How To Review`
  - 검수 방법과 표준 taxonomy 목록
- `Accident Review`
  - 아직 `기타`로 남은 사고유형 후보
- `Process Review`
  - 아직 `기타`로 남은 작업공정 후보

## 핵심 컬럼 설명

- `review_raw_value`
  - 실제 원문 표현입니다. 가장 먼저 봐야 하는 값입니다.
- `occurrence_count`
  - 이 표현이 몇 번 나왔는지입니다. 큰 것부터 검수하면 효과가 큽니다.
- `top_industries`
  - 이 표현이 많이 나온 업종입니다.
- `top_jobs`
  - 이 표현이 많이 나온 직종입니다.
- `sample_*`
  - 실제 문맥 예시입니다. 애매할 때 이 컬럼을 같이 보면 됩니다.
- `proposed_standard_group`
  - 당신이 최종적으로 넣을 표준 분류값입니다.
- `review_status`
  - 검수 완료 여부를 적는 칸입니다.
- `review_note`
  - 애매한 이유, 예외 처리 메모를 적는 칸입니다.

## 당신이 실제로 해야 할 일

1. `taxonomy_review_workbook.xlsx`를 엽니다.
2. `Accident Review` 시트에서 `occurrence_count`가 큰 순서대로 봅니다.
3. `review_raw_value`와 `sample_accident_type_raw`, `sample_narrative_text`를 같이 읽습니다.
4. 분류가 가능하면 `proposed_standard_group`에 표준 사고유형을 적습니다.
5. 검수가 끝나면 `review_status`에 `approved`를 적습니다.
6. 분류가 애매하면 `review_status`에 `skip`을 적고, 이유를 `review_note`에 남깁니다.
7. 같은 방식으로 `Process Review` 시트도 검수합니다.

### 공정 표준값

## 판단 기준

### 사고유형

- 다치게 된 직접 형태를 기준으로 봅니다.
- 진단명만 보여도, 다친 방식이 드러나면 그 방식으로 넣습니다.
- 예:
  - `골절`, `타박상`, `열린 상처`가 주로 부딪힘/맞음 맥락이면 `충돌`
  - `염좌`, `회전근개`, `반달연골`, `허리 통증` 계열이면 `무리한동작`
  - `절단`, `압궤`, `으깸`이면 `끼임`
  - `화상`이면 `화재폭발`
  - 차량, 배달, 출퇴근이면 `교통/이동`
- 도저히 직접 형태를 모르겠으면 `기타`로 둡니다.

### 공정

- 직종명 자체보다 “무슨 작업을 하는 상황인지”를 기준으로 봅니다.
- 예:
  - `택배`, `배달`, `화물차주`, `상하차`면 `상하차/물류`
  - `건설관련 기능`, `구조물 설치`, `비계`, `해체`면 `설치/해체`
  - `기계조작`, `가공`, `판금`, `금형`, `주조`면 `절단/가공`
  - `청소`, `조리`, `위생`, 정리 업무면 `청소/정리`
  - `검침`, `검사`, `시험`, 확인 업무면 `점검/검사`
  - `전기`, `배선`, `수전`, 분전반이면 `전기작업`
- 직종만 있고 작업 장면이 너무 넓으면 `기타`로 둡니다.

## 추천 검수 순서

- 1차:
  - 사고유형 상위 100개
  - 공정 상위 100개
- 2차:
  - `occurrence_count` 50 이상인 항목 전부
- 3차:
  - 나머지 롱테일 항목

## 검수할 때 주의할 점

- 억지로 분류하지 않아도 됩니다.
- 확신이 낮으면 `skip` 또는 `기타`가 더 낫습니다.
- 한 번 정한 기준은 비슷한 표현에 일관되게 적용하는 게 중요합니다.
- `review_note`에 기준을 남겨두면 다음 반영이 쉬워집니다.

## 검수 후

- 수정한 파일을 그대로 전달하면 됩니다.
- 가장 좋은 방식은:
  - `taxonomy_review_workbook.xlsx`에서 검수
  - 저장
  - 그 파일을 기준으로 다음 단계 반영

검수가 끝나면 그 결과를 제가 전처리 스크립트에 반영해서 3차 taxonomy 개선까지 이어갈 수 있습니다.
