# 공공데이터 기반 AI 작업 전 안전위험 분석 서비스의 구현 가능성 검토

## 1. 요약 결론

2026년 4월 기준으로 보면, 이 아이디어는 충분히 구현 가능한 편에 속합니다. 다만 지금 바로 구현 가능한 것은 특정 현장의 당일 사고발생 절대확률을 정밀하게 예측하는 단일 모델이 아니라, 오늘 작업 전 입력된 조건을 바탕으로 주요 사고유형과 중대 위험요인을 우선순위화하고, 유사 재해사례와 안전기준을 근거로 체크리스트와 브리핑 문안을 자동 생성하는 하이브리드 위험분석 서비스입니다.

즉시 가능한 범위는 다음과 같습니다.

- 사고유형 우선순위화
- 중대 위험요인 정리
- 유사 재해사례 검색 및 요약
- MSDS·날씨 반영 체크리스트 생성
- 현장 브리핑 문안 생성

반대로 공공데이터만으로 특정 현장의 당일 사고발생 절대확률을 정밀하게 예측하는 모델까지 바로 구현하는 것은 어렵습니다. 공개된 산재 데이터, 재해사례 API, SIF 아카이브, MSDS API, 날씨 예보·관측 API는 이미 존재하지만, 단기 예측의 핵심인 무사고 작업정보, near-miss, 작업허가, 현장 안전점검, 인원·공정 진척 같은 leading indicator는 공공데이터에 거의 없기 때문입니다.

## 2. 구현 가능성 판정

이 문서의 결론은 `조건부 구현 가능`입니다.

- 데이터 측면에서는 근로복지공단의 최초요양 승인 정보, 한국산업안전보건공단의 국내재해사례·사고사망 게시판·SIF 아카이브·MSDS API, 고용노동부의 산업재해 공표자료, 기상청의 예보·관측 API가 이미 공개되어 있어 MVP를 만들 재료는 갖춰져 있습니다.
- 연구 측면에서도 국가 산재 데이터, 프로젝트 데이터, 텍스트 사고보고서, 기상정보, 안전점검 기록을 활용한 예측·분류 모델의 실증 사례가 이미 축적되어 있습니다.

정리하면 다음과 같습니다.

- 가능한 것: 작업 전 위험 해석, 사고유형 우선순위화, 사례 검색, 브리핑 생성
- 내부 데이터가 있어야 가능한 것: 현장 단기 위험도 예측
- 지금 공공데이터만으로 곤란한 것: 특정 현장·작업자 수준의 정밀 사고확률, 법적 위험성평가 대체 자동판정

## 3. 데이터 준비 상태

### 3.1 구조화 데이터

구조화 데이터의 준비 상태는 비교적 양호합니다.

- 최초요양 승인 정보
  - 사업장 소재지, 업종명, 재해발생형태, 상병코드, 세부 상병명, 재해일자, 직종명, 성별, 연령대 등 포함
- 산업재해통계 마이크로 데이터
  - 재해자구분, 대·중업종, 발생형태, 규모, 성별, 연령대, 근무기간, 재해정도, 지역, 건설공사금액, 질병종류 등 포함

여기에 직업 코드, 기인물 코드, 상해종류 코드 같은 공식 분류체계를 결합하면 사고유형, 원인, 상해 결과에 대한 학습 라벨 공간을 비교적 일관되게 만들 수 있습니다.

### 3.2 비정형 데이터

비정형 데이터도 충분합니다.

- 국내재해사례 게시판 API
  - 업종 게시판 종류와 키워드로 사례 조회 가능
- 첨부파일 API
  - board 번호별 파일명과 다운로드 경로 제공
- 사고사망 게시판 API
  - 사망사고 일자, 장소, 사고개요, 사망자·부상자 규모 제공
- SIF 아카이브
  - 2016~2023년 중대재해 사례 6,032건 기반 고위험작업·상황, 재해유발요인, 위험성 감소대책 예시 제공
- MSDS API
  - 물질안전보건자료 목록 및 내용 제공
- 단기예보 API, ASOS 시간자료 API
  - 예보와 장기 시간별 관측값 제공

### 3.3 실제 난점

가장 큰 난점은 `작업공정`의 표준화입니다. 공공데이터에는 업종, 직업, 발생형태, 기인물, 상해종류 같은 공식 코드가 잘 갖춰져 있지만, 산업 전반을 관통하는 통합 작업공정 코드체계는 바로 제공되지 않습니다.

따라서 초기에는 `산업군-직종-주요 공정군` 수준의 자체 공정 taxonomy를 설계하고, 데이터가 풍부한 업종부터 좁혀 시작하는 방식이 현실적입니다.

또 하나 중요한 점은 데이터 갱신주기와 시차가 서로 다르다는 것입니다. 승인 데이터와 연간 통계는 시차가 크고, 사례 게시판과 MSDS, 기상 데이터는 훨씬 빠르게 갱신됩니다. 이 차이를 감안하면 당일 판단에는 실시간성 높은 소스를, 기준 학습과 벤치마크에는 연간 통계형 소스를 쓰는 이원화 전략이 적합합니다.

추가로 첨부파일 기반 비정형 데이터는 본문 파싱 품질이 실제 검색 품질을 크게 좌우합니다. 같은 사례라도 PDF 본문 추출 품질이 낮으면 유사사례 검색과 근거 요약의 정확도가 함께 떨어질 수 있으므로, RAG 계층에서는 파싱 실패를 별도 상태로 관리하고 메타만 유지하는 전략이 필요합니다.

## 4. 무엇을 예측해야 하는가

이 아이디어의 핵심은 하나의 모델이 모든 것을 맞히는 구조가 아닙니다. 공개 연구와 실제 데이터 제약을 감안하면 문제를 다음 네 가지로 나눠 보는 편이 타당합니다.

1. 어떤 사고유형이 우선 위험한가를 맞히는 다중분류 또는 다중라벨 문제
2. 이번 작업이 중대 위험군에 가까운가를 판단하는 위험등급화 문제
3. 지금 조건과 유사한 사고사례·MSDS·날씨 위험요인을 찾는 검색 문제
4. 그 결과를 행동문으로 바꾸는 생성 문제

이 네 문제는 각각 사용하는 데이터와 평가 방식이 다르므로, 아키텍처도 분리해서 설계해야 합니다.

공공데이터만으로는 `사고발생 절대확률`보다 `상대적 위험등급`이 더 타당한 목표입니다. 공개된 승인 데이터와 사고 데이터는 대부분 이미 사고가 발생했거나 승인된 사례 중심이고, 사고가 없었던 수많은 작업일, 작업허가, 점검기록은 포함하지 않기 때문입니다.

따라서 초기 서비스에서 `고위험`, `주의`, `표준관리` 같은 보정된 위험밴드와 근거 설명을 제시하는 것은 타당하지만, `오늘 사고확률 17.3%` 같은 숫자를 내세우는 것은 신뢰성과 책임 측면에서 과도합니다.

## 5. 권장 모델 구조

가장 현실적인 구조는 `예측 모델 + 검색 모델 + 생성 모델`을 결합한 하이브리드입니다.

### 5.1 구조화 입력 기반 분류 계층

입력값 예시:

- 업종
- 직종
- 작업공정군
- 작업장소
- 날짜·시간
- 기상요소
- 유해화학물질 여부
- 사업장 규모
- 근속기간

이 계층은 사고유형 후보와 위험등급을 우선 산출하는 용도로 사용합니다.

### 5.2 검색증강 계층

다음 자료를 대상으로 RAG 계층을 구성합니다.

- 국내재해사례 게시판
- 첨부파일 텍스트
- 사고사망 게시판
- SIF 아카이브
- MSDS 텍스트

이 계층은 현재 작업조건과 유사한 사례, 위험요인, 대응조치 문장을 검색하는 역할을 맡습니다.

### 5.3 생성 계층

생성형 AI는 다음 출력을 담당합니다.

- 위험요약
- 핵심 위험요인 설명
- 체크리스트 초안
- 작업 전 브리핑 문안
- 주의사항 요약

초기 기술선택은 `거대한 단일모델`보다 `경량 분류기 + 사례검색 + 근거기반 생성`이 더 실용적입니다.

## 6. 출력 문서 설계

산업안전보건법상 위험성평가는 사업주가 작업장의 유해·위험요인을 찾아내고 위험성의 크기를 평가한 뒤 필요한 조치를 해야 하며, 그 결과를 기록·보존해야 합니다. MSDS 대상물질을 취급하는 경우에는 물질안전보건자료 게시와 작업공정별 관리요령 게시 의무도 존재합니다.

따라서 AI가 만드는 출력은 단순 문장 나열보다 구조화 양식에 가까워야 합니다.

권장 출력 항목:

- 유해·위험요인
- 예상 사고유형
- 즉시 점검항목
- 필수 보호구
- 화학물질 주의사항
- 기상 경고
- 유사 사례 근거
- 작업 전 브리핑 핵심문장
- 최종 승인자 확인란

## 7. 평가 방식

평가도 하나의 점수로 끝내지 말고 작업별로 나눠야 합니다.

- 사고유형 예측: weighted F1-score
- 치명도·중대도 구분: AUROC 등 불균형 분류 지표
- 단기 위험경보: 시계열 슬라이딩 윈도 검증, threshold calibration
- 문서 생성: 안전관리자 검수 통과율, 수정률, 누락 핵심 점검항목 비율, 현장 피드백

즉, 이 서비스의 품질은 일반 NLP 점수보다 `실무 적용성`으로 평가하는 편이 적합합니다.

MVP 단계에서는 아래와 같은 운영 지표를 함께 두는 것이 좋습니다.

- gold set 기반 사고유형 Top N 일치율
- 관리자 1차 승인율
- 수정 후 승인 비율
- 치명 항목 누락률
- 검색 지표 예시: Recall@10, nDCG@10
- 생성 지표 예시: 근거 없는 문장 비율, 브리핑 실무성 평가

권장 방식은 업종별 100~150건 수준의 전문가 검수 gold set을 먼저 만들고, 여기에 `상위 사고유형`, `필수 점검 5개`, `작업중지 조건`, `MSDS/날씨 관련 필수문장`을 라벨링해 분류·검색·생성을 함께 평가하는 것입니다.

## 8. 제약과 위험

### 8.1 법적 대체 불가성

위험성평가는 법적으로 사업주의 의무이며, 근로자 참여와 결과 기록·보존이 요구됩니다. 따라서 이 서비스의 포지셔닝은 `위험성평가 자동화`가 아니라 `위험성평가 초안 생성 및 사전 브리핑 지원`이어야 합니다.

### 8.2 라이선스와 원문 사용범위

일부 데이터는 이용허락범위 제한이 없지만, SIF 아카이브처럼 출처표시·변경금지 조건이 붙는 자료도 있습니다. 또한 MSDS 정보는 참고용이라는 단서가 있습니다. 따라서 SIF와 MSDS는 주 학습데이터로 무제한 흡수하기보다, 원문 근거를 조회하고 인용 출처를 남기는 참고 자료로 다루는 편이 안전합니다.

### 8.3 날씨 데이터 정밀도

공개 API만으로도 단기예보와 ASOS 관측값을 결합해 MVP를 만드는 것은 충분하지만, 매우 세밀한 현장 미세기상 반영이 필요해지면 민간 서비스나 별도 센서 연동을 검토해야 합니다.

### 8.4 데이터 편향과 시차

승인 데이터는 승인 지연과 선택 편향이 있을 수 있고, 통계 데이터는 공개 시차가 큽니다. 텍스트 사고 데이터는 정성 서술, 혼합 변수, 주관적 기록, 누락정보 때문에 전처리와 표준화가 어렵습니다. 또한 첨부문서 파싱 품질이 낮으면 검색과 생성 품질도 함께 흔들릴 수 있습니다. 따라서 모델 출력에는 항상 예측근거와 확신도를 함께 보여주는 편이 바람직합니다.

## 9. 실무형 MVP 방향

MVP는 범용 산업 전체보다 건설·제조 중심의 좁은 범위로 시작하는 것이 좋습니다. 국내재해사례 게시판은 업종별 게시판이 구분되어 있고, SIF 아카이브도 건설·제조 데이터가 풍부합니다. 선행연구 역시 건설업에 상대적으로 많이 축적되어 있어 초기 모델 검증과 사용자 신뢰 확보에 유리합니다.

초기 버전에서 공공데이터만으로도 다음 기능은 충분히 구현 가능합니다.

- 유사 사례 검색
- 상위 사고유형 제시
- 중대 위험요인 정리
- SIF·MSDS·날씨 경고 결합
- 체크리스트와 브리핑 초안 생성

2단계에서 현장 안전점검, near-miss, 작업허가, 인원수, 하도급 정보, 공정 진척, 장비 점검, PPE 미준수 기록이 추가되면, 그때부터는 더 강한 단기 위험도 예측 엔진으로 발전시킬 수 있습니다.

운영 초기에는 `정적 체크리스트`와 `AI 생성 체크리스트`를 비교하는 A/B 테스트도 유용합니다. 특히 브리핑 준비시간 단축, 관리자 승인율, 치명 위험 누락률, 재사용률은 공공데이터 기반 서비스가 실제 현장 효율을 얼마나 높였는지 확인하는 핵심 지표가 됩니다.

## 10. 최종 판단

이 아이디어의 AI 구현은 가능합니다. 다만 성공하는 제품 정의는 `사고발생 절대확률 예측기`가 아니라 `공공데이터와 현장 입력을 결합한 작업 전 위험도 엔진`입니다.

정리하면, 지금 바로 만들 수 있는 것은 다음입니다.

- 위험유형 우선순위화
- 근거사례 검색
- 실행문서 생성

정밀한 단기 사고예측은 내부 leading indicator가 쌓인 뒤 2단계로 가는 것이 맞습니다. 이 방향으로 범위를 조정하면 기술적 실현가능성과 현장 적용가능성을 동시에 높일 수 있습니다.
