# Today Safety RAG 수집 작업목록

## 1. 목적

이 문서는 Today Safety RAG 코퍼스 1차 구축을 위한 구현 작업 순서와 완료 기준을 정리한 작업 목록입니다.

대상 범위는 다음으로 고정합니다.

- 산업: `건설 + 제조`
- 첨부파일: `PDF 본문 추출`, `HWP/HWPX 메타만 저장`
- 저장: `JSONL 본문 + PostgreSQL 메타`

## 2. 구현 순서

### 단계 1. 수집기 인터페이스 정의

작업:

- API/파일별 수집기 공통 함수 시그니처 정의
- `RawCase`, `RawAttachment`, `ExtractedDocument`, `CorpusDocument` 타입 초안 정의
- dataset id 기준 source 구분 enum 또는 상수 정의

완료 기준:

- 각 소스별 입력/출력 형태가 한 문서로 정리됨
- 구현자가 추가 의사결정 없이 수집기 파일을 나눌 수 있음

### 단계 2. 공통 문서 스키마 정의

작업:

- JSONL 스키마 필드 고정
- PostgreSQL 메타 컬럼 고정
- `doc_id` 규칙 고정
- `ingestion_status` 값 집합 정의

완료 기준:

- 모든 소스가 같은 `CorpusDocument` 구조로 변환 가능
- JSONL과 메타 CSV 간 연결 키가 `doc_id`로 확정됨

### 단계 3. 건설+제조 필터 규칙 정의

작업:

- 현재 전처리 taxonomy 기준으로 건설/제조 포함 규칙 정의
- 게시판/첨부/SIF/MSDS별 필터 적용 위치 정의
- 필터 실패 또는 불명확 데이터 처리 정책 정의

완료 기준:

- 어떤 문서가 1차 코퍼스에 포함되는지 구현자가 바로 판단 가능

### 단계 4. API 원문 수집기 설계

작업:

- `15121001` 게시판 본문 수집기 설계
- `15121008` 첨부파일 메타 수집기 설계
- `15119137` 사고사망 게시판 수집기 설계
- `15001197` MSDS 조회 방식 정리

완료 기준:

- 소스별 요청 파라미터, 페이지네이션, 저장 포맷이 정의됨

### 단계 5. PDF 다운로드 및 본문 추출기 설계

작업:

- PDF 다운로드 경로 규칙 정의
- PDF 본문 추출 모듈 정의
- 추출 실패/빈 본문/암호화 PDF 처리 기준 정의
- HWP/HWPX 메타 저장 포맷 정의

완료 기준:

- PDF는 본문 추출 가능/실패 모두 기록 가능
- HWP/HWPX는 링크와 메타를 잃지 않음

### 단계 6. normalize/enrich 설계

작업:

- 게시글 본문 -> `CorpusDocument`
- PDF 추출 본문 -> `CorpusDocument`
- SIF 행 -> `CorpusDocument`
- taxonomy 재사용 규칙 정의
- `text_summary` 생성 규칙 정의

완료 기준:

- source_type이 달라도 동일 스키마로 합쳐질 수 있음

### 단계 7. export 설계

작업:

- `corpus.jsonl` 출력 규칙 정의
- PostgreSQL 메타 CSV 또는 seed 파일 출력 규칙 정의
- 수집 실패 리포트 포맷 정의

완료 기준:

- 재생성 가능한 코퍼스 산출물이 명확함
- 실패 문서도 추적 가능함

## 3. 데이터셋별 작업 단위

### `15121001` 국내재해사례 게시판

- 게시글 목록/본문 수집
- board id 또는 게시글 식별자 추출
- 제목, 본문, 날짜, URL 저장
- 건설/제조 필터 적용

### `15121008` 재해사례 첨부파일 메타

- 게시글별 첨부 존재 여부 수집
- 파일명, 링크, 확장자 저장
- PDF만 다운로드 대상에 포함
- HWP/HWPX는 메타만 저장

### `15119137` 사고사망 게시판

- 게시글 1건을 문서 1개로 저장
- 날짜/장소/사고개요 중심 본문 구성
- severity는 높은 우선순위 문서로 태깅

### `15140383` SIF 아카이브

- 기존 전처리 구조 재사용
- 사례 1행 = 문서 1개
- `text_content`는 재해개요 + 유발요인 + 감소대책 조합으로 구성

### `15001197` MSDS

- 물질 1개 = 문서 1개
- 물질명, 위험요약, 주의사항을 코퍼스 문서로 구성
- 화학물질 관련 작업 보강용으로 사용

### `15084084` 단기예보

- 영구 코퍼스 저장 대상 아님
- 요청 시점 문맥 보강용 문서 생성 규칙만 정의

## 4. 테스트 및 검증

### 수집 테스트

- 게시글 수집 성공
- 첨부파일 메타 연결 성공
- PDF 다운로드 성공/실패 기록

### 정규화 테스트

- 게시글, SIF, 사고사망, PDF 추출문이 같은 스키마로 변환됨
- 건설/제조 외 문서가 기본 코퍼스에서 제외됨

### 저장 테스트

- JSONL 모든 문서에 필수 필드 존재
- 메타 CSV의 `doc_id`가 JSONL과 일치

### 실패 시나리오

- API 응답 누락
- 첨부 없는 게시글
- PDF 파싱 실패
- HWP/HWPX 첨부 존재

## 6. 완료 정의

이번 단계는 아래 조건을 만족하면 완료로 봅니다.

- 건설+제조 기준 코퍼스를 재생성할 수 있음
- PDF 추출 실패 문서도 메타가 유지됨
- 게시글 본문, PDF 본문, SIF, 사고사망이 공통 `CorpusDocument` 스키마로 조회 가능함
