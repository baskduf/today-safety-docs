# 공공데이터 기반 AI 작업 전 위험 브리핑 생성 서비스

## 1. 아이템 개요

본 서비스는 산업현장의 작업 조건을 입력하면, 공공데이터와 유사 재해사례를 바탕으로 AI가 오늘 작업의 핵심 위험요약, 주요 사고유형, 작업 전 체크리스트, 현장 브리핑 문안을 생성하는 서비스이다.

핵심은 `정밀 사고확률 예측`이 아니라, 공공데이터에 축적된 사고사례와 위험요인을 오늘 작업에 맞는 실행 문서로 전환하는 데 있다.
또한 본 서비스는 법적 위험성평가를 자동 대체하는 시스템이 아니라, `위험성평가 초안 생성 및 작업 전 사전 브리핑 지원 도구`로 포지셔닝하는 것이 타당하다.

## 2. 제안 배경

산업현장에서는 안전점검표가 존재하더라도 실제 작업 조건을 충분히 반영하지 못해 형식적으로 활용되는 경우가 많다. 같은 작업이라도 산업군, 직종, 작업공정, 작업장소, 날씨, 화학물질 사용 여부에 따라 위험요인이 달라지지만, 현장에서는 이를 실시간으로 반영한 브리핑과 점검 항목을 매번 도출하기 어렵다.

특히 중소사업장과 다품종 작업환경에서는 안전관리 인력과 시간이 부족해 작업 전 위험요인을 충분히 분석하지 못하는 문제가 있다. 그 결과 안전정보는 존재하지만, 실제 작업 시작 전에 곧바로 실행 가능한 형태로 전달되지 못한다.

## 3. 문제 정의

- 정적인 안전점검표로는 당일 작업 조건 반영이 어렵다.
- 현장관리자가 방대한 안전자료와 재해사례를 직접 해석해야 한다.
- 작업 전 브리핑이 경험에 의존해 일관성이 낮다.
- 위험은 인지하고 있어도 이를 행동 가능한 점검 항목과 브리핑 문장으로 바꾸는 과정이 비효율적이다.

## 4. 해결 방안

사용자가 산업군, 직종, 작업공정군, 작업장소, 기상조건, 유해요인 정보를 입력하면 시스템이 공공데이터와 외부 환경데이터를 조회한다. 이후 AI가 주요 사고유형과 중대 위험요인을 정리하고, 유사 재해사례와 안전기준을 근거로 작업 전 체크리스트와 브리핑 문안을 자동 생성한다.

즉, 본 서비스는 단순히 안전정보를 검색해 주는 것이 아니라, 위험분석 결과를 현장에서 바로 활용 가능한 실행 문서로 변환해 주는 AI 서비스이다.

## 5. 주요 기능

### 1차 우선 기능

- 오늘 작업 위험요약 생성
- 주요 사고유형 Top N 우선순위 제시
- 중대 위험요인 정리
- 유사 재해사례 자동 검색 및 요약
- 작업 전 안전 체크리스트 자동 생성
- 현장 반장 및 관리자용 브리핑 문안 자동 생성
- 화학물질 사용 시 MSDS 기반 주의사항 반영
- 날씨 변화 반영 위험 경고 제공

### 후속 고도화 기능

- 사용자 승인/수정 로그 기반 개인화
- 현장별 위험등급 보정
- 작업중지 검토 조건 정교화

## 7. AI 활용 방식

본 서비스에서 AI는 단순 문장 생성기가 아니라, `위험 해석 + 사례 검색 + 문서 생성`을 수행하는 실행지원 계층으로 활용된다.

- 사고사례, 산재승인정보, MSDS, 날씨, 작업조건을 종합해 오늘 작업의 핵심 위험을 정리
- 유사 사고유형과 중대 위험요인을 우선순위화
- 분석 결과를 바탕으로 체크리스트와 브리핑 문안 자동 생성
- 다량의 재해사례와 안전자료를 사람이 이해하기 쉬운 행동 문장으로 변환

기술 구조는 `경량 예측 모델 + 데이터 검색 + 생성형 AI`의 결합 형태를 기본으로 한다.

## 8. 서비스 이용 흐름

1. 사용자가 오늘의 작업 조건을 입력한다.
2. 시스템이 공공데이터와 날씨 데이터를 조회한다.
3. AI가 유사 사례와 핵심 위험요인을 정리한다.
4. 주요 사고유형, 체크리스트, 보호구, 브리핑 문안을 생성한다.
5. 현장관리자 또는 승인자가 결과를 검토하고 수정 또는 승인한다.
6. 사용자는 승인된 결과를 작업 전 점검 및 현장 브리핑에 활용한다.

## 9. 차별성

- 단순 안전정보 검색이 아니라 작업 전 실행 문서 생성에 초점을 둔다.
- 단순 조건 분기형 점검표가 아니라 유사 사례와 공공데이터를 함께 활용한다.
- 정적인 점검표가 아닌 당일 작업조건 반영형 서비스이다.
- 사고유형 정리, 사례 근거, 브리핑 생성을 함께 제공한다.
- 다양한 산업현장에 확장 가능한 범용 구조를 가진다.

## 10. 기대효과

- 형식적 안전점검을 실질적 사전 예방 활동으로 전환
- 작업 전 위험 인지 수준 향상
- 중소사업장의 안전관리 준비시간 단축
- 현장 브리핑의 표준화 및 일관성 확보
- 공공데이터의 실사용 가치 확대

## 11. 실현 가능성

본 서비스는 이미 개방된 공공데이터를 기반으로 구현 가능하며, 초기에는 고정밀 사고확률 예측보다 `작업 전 위험 해석과 브리핑 생성` 중심으로 MVP를 개발하는 것이 현실적이다. 특히 데이터가 풍부하고 위험 시나리오가 선명한 `건설·제조`를 우선 산업군으로 삼는 편이 구현과 검증 모두에 유리하다.

공개 공공데이터만으로 모든 산업군에서 실시간 정밀 사고확률을 예측하는 것은 어렵지만, 작업조건 기반 위험요약, 사고유형 우선순위화, 유사 재해사례 기반 설명, 체크리스트와 브리핑 생성은 충분히 설계 가능하다. 따라서 공모전 단계에서는 `정밀 예측 AI`보다 `공공데이터 기반 위험분석 결과를 행동 가능한 안전문서로 전환하는 AI`라는 포지션이 가장 현실적이다.

자세한 구현 가능성 검토는 [implementation_feasibility_review.md](./implementation_feasibility_review.md)에 정리하였다.

## 12. 결론

본 아이디어는 공공데이터를 단순 열람하는 수준을 넘어, 실제 산업현장에서 즉시 활용 가능한 안전행동으로 전환하는 데 목적이 있다. 공공데이터와 AI를 융합해 오늘 작업의 주요 위험을 정리하고, 이를 체크리스트와 브리핑 문안으로 자동 생성함으로써 산업현장의 사고예방과 안전문화 정착에 기여할 수 있다.

## 13. 구현 전략 요약

초기 MVP는 모든 산업군에 대한 정밀 사고확률 예측을 목표로 하지 않는다. 대신 작업조건 기반 위험요약, 주요 사고유형 도출, 유사 재해사례 기반 설명, 체크리스트와 브리핑 생성에 집중한다.

이를 통해 공공데이터의 한계를 벗어나지 않으면서도, 실제 현장에서 체감 가능한 결과물을 제공하는 방향으로 구현한다.

## 14. 단계별 로드맵

1. 데이터 정규화
   - 공공데이터 수집 구조 정리, taxonomy 정규화, 사례/기상/MSDS 연결 기반 구축
2. 검색 중심 MVP
   - 건설·제조 중심 유사사례 검색, 체크리스트 생성, 브리핑 생성, 승인 UI 초안 구현
3. 하이브리드 분류·등급화
   - 사고유형 Top N 분류기와 상대 위험등급 보정 계층 추가
4. 파일럿 및 로그 기반 고도화
   - 승인·수정 로그, 현장 피드백, 운영 데이터를 바탕으로 개인화와 위험도 보정 강화

## 15. 권장 기술 구성

- 프론트엔드: Next.js, TypeScript
- 백엔드: FastAPI, Pydantic
- AI 계층: Python 백엔드 내부 서비스 모듈
- 데이터 저장소: PostgreSQL
- 외부 AI 연동: OpenAI Responses API

초기 단계에서는 프론트엔드와 백엔드를 분리하되, AI는 별도 독립 서비스로 떼지 않고 백엔드 내부 계층으로 포함하는 방식이 가장 현실적이다. 이 구조는 공모전 데모 완성도, 개발 속도, 운영 단순성 측면에서 유리하다.
